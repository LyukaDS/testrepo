Як ми торкалися в одному з наших попередніх відео, джерела даних ніколи не були такими динамічними та різноманітними, як сьогодні. У цьому відео ми розглянемо деякі поширені джерела, такі як реляційні бази даних; Плоскі файли та набори даних XML API та веб-служби; Веб-скрапінг; Потоки даних та канали Як правило, організації мають внутрішні програми для підтримки їх у управлінні повсякденною діловою діяльністю, транзакціями клієнтів, діяльністю людських ресурсів та їхніми робочими процесами. Ці системи використовують реляційні бази даних, такі як SQL Server, Oracle, MySQL та IBM DB2, для зберігання даних структурованим способом. Дані, що зберігаються в базах даних і сховищах даних, можуть бути використані як джерело для аналізу. Наприклад, дані системи роздрібних транзакцій можуть бути використані для аналізу продажів у різних регіонах, а дані системи управління відносинами з клієнтами можуть бути використані для складання прогнозів продажів. За межами організації існують інші загальнодоступні та приватно доступні набори даних. Наприклад, державні організації випускають набори демографічних та економічних даних на постійній основі. Потім є компанії, які продають конкретні дані, наприклад, дані торгових точок або фінансові дані, або дані про погоду, які підприємства можуть використовувати для визначення стратегії, прогнозування попиту та прийняття рішень, пов'язаних з розповсюдженням або маркетинговими акціями, серед іншого. Такі набори даних зазвичай доступні у вигляді плоских файлів, файлів електронних таблиць або XML-документів. Плоскі файли зберігають дані у звичайному текстовому форматі, з одним записом або рядком на рядок, і кожне значення розділене роздільниками, такими як коми, напівкрапки або вкладки. Дані в плоскому файлі відображаються в одній таблиці, на відміну від реляційних баз даних, які містять кілька таблиць. Одним з найпоширеніших форматів плоских файлів є CSV, в якому значення розділені комами. 

Файли електронних таблиць — це особливий тип плоских файлів, які також організовують дані у табличному форматі — рядки та стовпці. Але електронна таблиця може містити кілька аркушів, і кожен аркуш може відображатися з іншою таблицею. Хоча дані в електронних таблицях є звичайним текстом, файли можуть зберігатися у користувацьких форматах і містити додаткову інформацію, таку як форматування, формули тощо.

 Microsoft Excel, який зберігає дані у форматі.XLS або .XLSX, ймовірно, є найпоширенішою електронною таблицею. 
Інші включають таблиці Google, номери Apple та LibreOffice. 

Файли XML містять значення даних, які ідентифікуються або позначені за допомогою тегів. 

Хоча дані в плоских файлах є «плоскими» або відображаються в одній таблиці, XML-файли можуть підтримувати більш складні структури даних, такі як ієрархічні. 
Деякі поширені способи використання XML включають дані з онлайн-опитувань, банківських виписок та інших неструктурованих наборів даних. 


Багато постачальників даних та веб-сайти надають API, або інтерфейси прикладних програм та веб-сервіси, з якими можуть взаємодіяти кілька користувачів або додатків та отримувати дані для обробки або аналізу. 
API та веб-служби зазвичай прослуховують вхідні запити, які можуть бути у формі веб-запитів користувачів або мережевих запитів від програм, і повертають дані у простому тексті, XML, HTML, JSON або медіафайлах. 

Давайте розглянемо деякі популярні приклади використання API як джерела даних для аналітики даних: 
використання API Twitter та Facebook для отримання даних із твітів та публікацій для виконання таких завдань, як видобуток думок або аналіз настроїв, що полягає в узагальненні кількості вдячності та критики щодо певної теми, 
наприклад політики уряду, продукту , послуги або задоволеності клієнтів загалом. 
API фондового ринку використовуються для отримання даних, таких як ціни на акції та товари, прибуток на акцію та історичні ціни, для торгівлі та аналізу. 
API пошуку та перевірки даних, які можуть бути дуже корисними для аналітиків даних для очищення та підготовки даних, а також для спільного зв'язку даних, наприклад, для перевірки того, до якого міста чи штату належить поштовий або поштовий індекс. 
API також використовуються для вилучення даних з джерел бази даних, як всередині організації, так і за її межами. 

Web ScrapingWeb Scraping використовується для вилучення відповідних даних з неструктурованих джерел. 
Також відомий як скребкування екрана, збирання веб-сторінок та вилучення веб-даних, 
веб-скребкування дозволяє завантажувати конкретні дані з веб-сторінок на основі визначених параметрів. 

Веб-скрепери можуть, серед іншого, витягувати текст, контактну інформацію, зображення, відео , елементи продукту та багато іншого з веб-сайту. 

Деякі популярні види використання веб-скреблінгу включають збір інформації про продукти від роздрібних продавців, виробників та веб-сайтів електронної комерції для порівняння цін; створення потенційних клієнтів за допомогою загальнодоступних джерел даних; 
вилучення даних з публікацій та авторів на різних форумах та спільнотах; 
а також збір навчальних та тестових наборів даних для моделей машинного навчання 
Деякі з популярних інструментів веб-скребкінгу включають BeautifulSoup, Scrapy, Pandas та Selenium. 

Потоки даних є ще одним широко використовуваним джерелом для агрегації постійних потоків даних, що надходять з таких джерел, як інструменти, пристрої IoT та додатки, дані GPS з автомобілів, комп'ютерних програм, веб-сайтів та публікацій у соціальних мережах. 

Ці дані, як правило, позначені часом, а також гео-позначені для географічної ідентифікації. Деякі потоки даних та способи їх використання включають: біржові та ринкові тикери для фінансової торгівлі; потоки роздрібних транзакцій для прогнозування попиту та управління ланцюгами поставок; спостереження та відеоканали для виявлення загроз; канали соціальних медіа для аналізу настроїв; канали даних датчиків для моніторингу промислової або сільськогосподарської техніки; веб-канали кліків для моніторингу продуктивності веб-сайтів та вдосконалення дизайну; та події польоту в режимі реального часу для перебронювання та перенесення. Деякі популярні програми, які використовуються для обробки потоків даних, включають Apache Kafka, Apache Spark Streaming та Apache Storm. RSS-канали (або дійсно проста синдикація) - ще одне популярне джерело даних. Зазвичай вони використовуються для збору оновлених даних з онлайн-форумів та новинних сайтів, де дані оновлюються на постійній основі. Використовуючи зчитувач каналів, який є інтерфейсом, який перетворює текстові файли RSS в потік оновлених даних, оновлення передаються на пристрої користувача.
